# Expense Tracker

Build an app to track the expense I make

_Generated by AI Foundry (NVIDIA Nemotron)._

## Using the app

### Dashboard
This is the main screen where you get an overview of your spending.
- **Total Expenses / This Month / Transactions cards:** Show your overall spending, spending for the current month, and the total number of recorded transactions.
- **Recent Expenses:** Lists your 5 most recent transactions, showing the description, category, date, and amount.
- **By Category:** Displays a breakdown of your spending per category with horizontal bars representing the total amount spent in each.

### Add Expense
This screen allows you to record a new transaction.
- **Description:** Type what the expense was for (e.g., "Groceries").
- **Amount ($):** Enter the cost of the expense.
- **Category:** Select a category from the dropdown (Food, Transport, Shopping, Bills, Health, Entertainment, Other).
- **Date:** Pick the date the expense occurred.
- **Save Expense:** Click this button to record the expense and return to the Dashboard.

### Edit Expense
This screen allows you to modify an existing transaction.
- **Form fields:** Edit the Description, Amount, Category, or Date just like on the Add Expense screen.
- **Save Expense:** Click to apply your changes and return to the History screen.
- **Cancel:** Click to discard any changes and go back to the History screen.

### History
This screen shows a list of all your past transactions.
- **Category filter buttons:** Click a category button (e.g., Food, Bills) to only show expenses for that category. Click "All" to see everything.
- **Search expenses:** Type a word into this box to filter expenses by their description.
- **Edit (pencil icon):** Click this on any expense row to go to the Edit Expense screen for that item.
- **Delete (trash icon):** Click this on any expense row, then confirm the popup, to permanently remove the transaction.

## Quick test
1. Open the app and confirm the **Dashboard** loads with "No expenses yet" and $0.00 totals.
2. If the app is configured with a backend (`window.API_BASE`), open your browser's developer console Network tab and confirm it successfully fetches from `/api/store/expenses` without errors.
3. Click **Add Expense** in the navigation, fill out the form (e.g., "Coffee", $4.50, Food, today's date), and click **Save Expense**.
4. Confirm you are returned to the Dashboard, and the "Total Expenses", "This Month", and "Transactions" cards now reflect the expense you just added.
5. Click **History** in the navigation, confirm your new expense appears in the list, then click the **Edit** (pencil) button for that expense, change the amount, click **Save Expense**, and verify the updated amount shows in the history list.
6. Still on the History screen, use the **Category** filter buttons and the **Search** box to confirm the list filters correctly.
7. Click the **Delete** (trash) button on the expense, confirm the popup, and verify the expense is removed from the list.

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
