## Using the app

The app has a single-page layout with two main views navigated via links at the top of the screen.

### Navigation
- **Dashboard** link: Switches to the summary view.
- **Expenses** link: Switches to the expense entry and list view.

### Dashboard
- **Total Expenses card**: Shows the total dollar amount and count of transactions currently displayed (affected by the category filter on the Expenses page).
- **Average per Transaction card**: Displays the average cost per transaction for the current filter.
- **Highest Single Expense card**: Displays the maximum single expense amount for the current filter.
- **Category Breakdown chart**: A horizontal bar chart appearing below the summary cards. It shows a bar for each category that has expenses, scaled relative to your highest category, with the total dollar amount next to it.

### Expenses
- **Add/Edit Expense form**: Contains fields for Description (text), Amount (number), Category (dropdown: Food, Transport, Entertainment, Utilities, Health, Other), and Date (date picker). 
  - **Add Expense button**: Submits the form and adds the new expense to the top of the list.
  - **Cancel button**: Appears only when editing an existing expense; clears the form and exits edit mode.
- **Category filter pills**: A row of clickable buttons (All, Food, Transport, etc.) at the top of the list. Clicking one filters the expense list and updates the Dashboard totals to only that category. "All" shows every expense.
- **Expense list**: Displays all added expenses matching the current filter. Each item shows its description, a colored category badge, the date, and the amount.
  - **Edit button (yellow)**: Loads that expense's details back into the form at the top for updating.
  - **Delete button (red)**: Immediately removes the expense from the list.

## Quick test
1. Click the **Expenses** link and fill out the Add New Expense form with a Description, Amount, Category, and Date; click **Add Expense** and confirm the new item appears at the top of the list.
2. Click the **Edit** button on that new expense, change the Description and Amount, click **Update Expense**, and confirm the item updates in the list.
3. Add a second expense with a different Category, then click that category's name in the filter pills above the list; confirm only expenses matching that category appear, and click **All** to bring back the full list.
4. Click the **Dashboard** link and confirm the Total Expenses, Average per Transaction, and Highest Single Expense cards reflect the correct calculated amounts.
5. Confirm the Category Breakdown bars appear, are proportionally sized, and display the correct dollar totals next to each category name.
6. Return to the **Expenses** tab, click **Delete** on one of the expenses, and confirm it disappears from the list and the Dashboard totals update accordingly.
7. Refresh the page in your browser and confirm your remaining expense(s) are still there (saved to local storage).

**Architecture:** frontend: React (Vite) + React Router; backend: FastAPI (Python); database: PostgreSQL (Render) / browser localStorage in demo; deployment: Docker containers -> Vercel (frontend) + Render (backend + Postgres)

## Run the full stack with Docker
```bash
docker compose up --build
```
- Frontend (nginx, Vite build): http://localhost:${FRONTEND_PORT}
- Backend (FastAPI / uvicorn): http://localhost:${BACKEND_PORT} (see `/docs`)

Host ports are written to `.env` by the platform when it brings the stack up.

## Run the backend directly (without Docker)
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
The API runs at http://localhost:8000 (see `/docs` for Swagger UI).

## Deploy to Render
```bash
# Launch the render.yaml Blueprint (web service + Postgres) from the
# Render dashboard, or build the image locally:
cd backend
docker build -t app .
```
